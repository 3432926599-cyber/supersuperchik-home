⭐ 星星工作流

五个 /命令，从想法到上线一口气跑完。
AI 不替你思考——替你执行。GitHub Stars 第一的工作流封装。

$ man 审讯式PRD

Agent 不会直接接受你的第一描述。它会像审讯官一样追问，把模糊点变成精准定义。

每个回答 → 术语定义沉淀 (CONTEXT.md) 每个决策 → 架构记录留档 (docs/adr/) 最终产出 → 精准 PRD + 完整边界声明
00
项目初始化 /setup-matt-pocock-skills
Agent 三问 — Issue Tracker / 标签体系 / 文档布局。回答即走，不做多余配置。
CONTEXT.md 术语表就位 + docs/adr/ 就绪
01
需求 → PRD /grill-with-docs
深度审讯式追问：帮你厘清思路，梳理需求，工作留痕。
PRD 发布到 GitHub Issue，标签 ready-for-agent
02
PRD → Issues /to-issues
垂直切片拆分 — 每个 Issue = Schema→API→UI→Test 全走通。
N 个子 Issue 可并行开发，标注 AFK / HITL
03
实现 & 测试 /tdd
Red 写失败的测试 → Green 最少代码过关 →Blue 重构优化。一次一个测试，红时不重构。
每 Issue 独立可验证交付，测试绿全覆盖
04
质量巡检 /improve-codebase-architecture
全库扫描 → HTML 可视化报告。浅模块、紧耦合、Before/After 对比图。
架构报告 + 重构建议，可逐项执行

# SKILL OVERVIEW TIER
★ 01 frontend-design 创建独特的、生产级前端界面，避 AI 塑料感。双轮迭代设计流程：构思→批判→编码，严格从设计计划推导每个色值和字体选择 GOLD
★ 02 deep-research 深度调研引擎 — 扇形展开多源搜索 → 交叉验证 → 合成带引用的完整报告。适合需要多维度、事实核查的复杂问题 GOLD
★ 03 code-review Diff 正确性审查 + 可简化性分析，支持低/中/高三级力度。可选 --comment 发布为 PR 行内评论或 --fix 直接应用到工作树 GOLD
★ 04 diagnose 六阶段结构化 Debug 循环：复现→最小化→假设→插桩→修复→回归测试。适用于难以定位的 Bug 和性能退化 GOLD
★ 05 improve-codebase-architecture 扫描全库，输出 HTML 可视化架构报告：模块深度分析、耦合检测、Before/After 对比图、优先级排序。防代码腐化 GOLD
◆ 06 grill-with-docs 深度需求拷问，边问边沉淀术语到 CONTEXT.md + 创建架构决策记录 (ADR)。确保需求与领域模型对齐 SILVER
◆ 07 to-prd 从对话上下文和代码库理解中合成完整 PRD，包含问题陈述→解决方案→User Stories→实现决策→边界声明。自动发布到 GitHub Issue SILVER
◆ 08 to-issues PRD 垂直拆分为可并行开发的曳光弹 Issues，每个 Issue = Schema→API→UI→Test 全走通，标注 AFK/HITL 类型 SILVER
◆ 09 tdd 红→绿→重构循环。一次只一个测试，只验证公共接口行为，不许测私有方法。先绿后重构，红时不重构 SILVER
◆ 10 verify 验证代码变更是否真正生效 — 运行应用并观察行为，确认修复/功能按预期工作 SILVER
◆ 11 handoff 压缩当前对话为交接文档，下一个 Agent 无缝接手。适合上下文过长需要换会话的场景 SILVER
◆ 12 simplify 审查变更代码的复用性、简化性、效率和高层抽象优化。纯质量审查，不猎 Bug（用 code-review） SILVER
• 13 update-config 配置 Claude Code harness settings.json — 权限、钩子、环境变量。支持项目级/用户级配置分离 BRONZE
• 14 loop 按间隔重复运行 prompt 或 slash 命令，支持自定义间隔。适用于轮询部署状态、定时检查等场景 BRONZE
• 15 run 启动并驱动项目应用，支持 CLI/Server/TUI/Electron/浏览器驱动等多种项目类型自动识别 BRONZE
• 16 claude-api Claude API / Anthropic SDK 参考 — 模型 ID、定价、参数、流式、工具使用、MCP、Token 计数、模型迁移 BRONZE
• 17 security-review 安全审查 — 对代码变更进行安全维度扫描，发现潜在安全漏洞和风险点 BRONZE
• 18 init / review 项目初始化 + PR 审查。init 搭建项目骨架，review 拉取并审查 Pull Request BRONZE
• 19 fewer-permission-prompts 扫描 transcripts 中的只读 Bash/MCP 工具调用，添加优先白名单到 .claude/settings.json，减少权限提示 BRONZE
